# Audit — juli 2026 (etter funksjonsbølgen)

Fersk gjennomgang etter alt som er bygget siden forrige audit: CORS, CSP (Report-Only),
«Min side», konto-bøttelagring, integritetsskala/POMP, rolleprofiler (revidert),
analytics, AI-samtykke, PWA, gruppert meny, fargeopprydding, modellbytte til
gemini-3-flash-preview. Metode: målrettet kodegjennomgang + npm audit + verifisering
av kjente invariante egenskaper (auth-dekning, idempotent kreditering, bøttelogikk).

## Bekreftet i orden (nytt siden sist)

- CORS låst til APP_URL; admin-endepunkt rate-limited; Stripe promo-codes aktivert.
- `/api/track`: allow-liste + rate-limit, aggregert per dag, ingen bruker-ID — i tråd
  med «100 % lokalt»-løftet. Kjøp telles idempotent inne i webhook-transaksjonen.
- Konto-bøttelagring: `bigfive_prep_score_history` og `bigfive_prep_ai_consent` er
  begge med i STORAGE_KEYS → følger backup/gjenoppretting OG slettes korrekt ved
  GDPR-sletting/reset. Logikken er scenario-testet (6 tilfeller).
- AI-samtykke gates begge AI-funksjonene; eksport/import og PDF påvirkes ikke.
- Kontrastfiks på gull-badges (WCAG ≥ 4.5:1).

## Funn

### A1 · HØY (5 min å fikse) — CSP står fortsatt i Report-Only
`server.ts:48` har `reportOnly: true` — «fase 2» ble aldri utført. Headeren beskytter
i dag ingenting; XSS-forsvaret er reelt sett av. **Anbefaling:** åpne live-siden,
F12 → Console, gjennomfør innlogging + kjøpsmodal + AI-analyse; er konsollen fri for
`[Report Only]`-brudd, flipp til `false` og deploy.

### A2 · MEDIUM — Preview-modell uten fallback
Begge AI-endepunktene bruker `gemini-3-flash-preview` (server.ts:593, 839). Google kan
endre/fjerne preview-modeller uten varsel → begge betalte funksjoner dør samtidig, og
fiksen krever ny build+deploy. **Anbefaling:** les modellnavn fra env-var
(`GEMINI_MODEL` med default), og/eller legg inn fallback til `gemini-2.5-flash` ved
404 fra modell-APIet.

### A3 · MEDIUM (venter på deg) — Salg kan ikke aktiveres ennå
Uendret siden sist: `[Selskapsnavn]`, org.nr, e-post og dato er ufylt i
PERSONVERN.md/VILKAR.md; `STRIPE_SECRET_KEY`/`STRIPE_WEBHOOK_SECRET` og
`RESEND_API_KEY`/`EMAIL_FROM` mangler i .env. Koden er klar; dette er de eneste
blokkerne for lovlig, fungerende salg.

### A4 · LAV — /api/track kan støyes av en motivert aktør
Uautentisert skriving (bevisst — anonyme skal telles), begrenset til 10/min/IP per
instans (×5 instanser). En bot kan blåse opp tallene/skape marginale skrivekostnader,
men kan ikke skrive vilkårlige felt. Akseptabel risiko for aggregert statistikk.

### A5 · LAV — Statistikk-døgnet følger UTC
`todayKey()` gir døgnskille kl. 01/02 norsk tid. Kosmetisk; bytt til Europe/Oslo-
formatering om tallene skal leses per norsk kalenderdag.

### A6 · LAV — npm audit: 6 moderate (uendret)
Transitiv `uuid` via firebase-admin; ikke utnyttbar i vår bruk; venter på upstream.

### Info
- In-memory rate-limiter per instans: kjent, bevisst akseptert (budsjett-tak er backstop).
- `server.ts` er nå ~900 linjer; vurder modulsplitting ved neste større serverarbeid.
- AI-samtykket håndheves kun klient-side. Greit: API-kalleren ER datasubjektet, og
  dermed kan ingen andres data sendes uten deres handling — men dokumentert her.
- Eldre audit-dokumenter (AUDIT-2026-06*) beskriver ikke ny funksjonalitet.
